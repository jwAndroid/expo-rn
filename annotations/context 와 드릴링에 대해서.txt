Context 에 대한 개념 그리고 드릴링

App 에서 부모 -> 자식 -> 자식의 부모 -> 자식의 자식의 부모 
의 어떤값을 제일 바깥에서 제일 처음 부모한테 넘길떄(props) 로 밑에 자식들은 계속해서 받아야함
이런걸 드릴링이라고하고

그런데 이렇게 드릴링을 안하고 바로 접근 가능한게 리덕스나 context 라고 하는데
context는 비동기처리가 안됨 그래서 theme , 다크모드(마음먹으면 redux로 가능) 활성화 할때 많이 쓴다고함.
context는 직접 코딩하는건 아니고 개념을 알아야한다.

중요한건 드릴링을 안하고 바로 접근할수 있다는것. 리덕스처럼

리랜더링 될떄?
프랍스나 스테이트랑 바뀌었을때

한부분에서만 데이터가 놀때 useState를 쓰고
넘어가는경우는 redux 를 쓸것

라이브러리 만들때 context 를 쓴다
네이게이션내부에 context 가 내장되어있다고 한다.
드릴링은 하면안되고 대신 redux로 관리해야한다.
